Key Highlight (Slide Sync)

1. APP_CREATED (app_expired)

Kendala pada tahap application inquiry. Catatan utama berada saat melakukan proses inquiry saca atau cif.

2. PAYROLL_APP (app_expired)

Kendala pada proses pengisian data dan inquiry data cif. Data pengajuan gagal diproses masuk ke scoring akibat berkas tidak lengkap.

3. KTP_REJ

Penawaran credit tidak diterima/ditolak atau foto KTP direject otomatis karena hasil tangkapan kamera tidak jelas atau data tidak sesuai NIK pada CIF.

4. DISB_ACC_CONF & DOCUMENT_SIGN

Verifikasi demografi atau face match gagal. Jika proses tanda tangan digital gagal, saat ini user dipaksa melakukan pengajuan ulang dari awal.

Actionable Insights (Apa yang Perlu Kita Lakuin?)

Optimasi Sistem & Whitelist Checking

Kebocoran terbesar ada di PAYROLL_REJ (5.939 kasus). Sebanyak 21.08% gagal karena WL Tidak Terdaftar. Perlu koordinasi integrasi data master karyawan.

Prioritas: Critical System

Perbaikan Fitur OCR & Panduan KTP

Sebanyak 233 berkas (2.17%) tertahan di KTP_REJ. UI aplikasi perlu ditambah kerangka penempatan kamera otomatis (auto-capture) agar foto tidak buram.

Prioritas: UX Improvement

Mekanisme Recovery Tanda Tangan

Saat tanda tangan digital gagal, user harus mengulang dari nol. Dashboard mengusulkan fitur "Resume Application" khusus tahap akhir.

Target: Mengurangi USR_REJECT

Evaluasi Durasi Masa Kedaluwarsa Engine

Terdapat penumpukan status app_expired (492 & 282 kasus) saat verifikasi scoring berjalan. Perlu cek latensi pengiriman antrean data ke biro.

Target: Memangkas Latency Scoring
